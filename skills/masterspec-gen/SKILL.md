# masterspec-gen — сгенерировать один артефакт

Узел-исполнение (`../masterspec/references/patterns/element-workflow.md`). Я генерирую РОВНО ОДИН артефакт указанного типа в изолированном контексте — не вижу всю фабрику, только поданный фокус-набор. Чужое — только через ссылку.

## Вход (изолированный фокус-набор, R-ISO)
Готовит оркестратор/explore: бизнес-описание ровно этого элемента + релевантный срез вышестоящих артефактов (для fn — cdm+rules+глоссарий; для cmp — fn; для api — cmp+data) + шаблон. НЕ подаётся остальная фабрика.

## Метод
1. Прочитай шаблон `../masterspec/templates/tpl-<type>.md`, скопируй структуру.
2. Заполни по мета-модели. Дисциплина слоёв — `../masterspec/references/layer-discipline.md` (запреты для слоя элемента).
3. Тип-специфика: fn — альтернативные/исключительные потоки обязательны, AC как оракул given-when-then; cdm/data — раздел «состояния и переходы» (матрица без пустых ячеек); api — обе стороны контракта (вход/ответ-успех/ответ-отказ), типы; nfr — числовая метрика + квалификаторы.
4. Ссылки `-> slug` только вверх/в слое.

## Дозапрос (≤2 раунда)
Не хватает фокус-набора (упомянута сущность вне поданного среза, неясное правило) — НЕ выдумывай. Запрос «не хватает X» → оркестратор → explore добирает. Лимит 2, потом «Открытый вопрос» человеку.

## Самопроверка перед отдачей
Прогон своего артефакта по оси 1 (`patterns/verification-axes.md`): пустой обязательный пункт → доделать или пометить «Открытый вопрос». Это НЕ замена verify слоя — лишь элементная гигиена.

## Выход
- `<type>-<slug>.md` (`status: draft`); список дозапросов и открытых вопросов → оркестратору в route-run.
