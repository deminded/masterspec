# Runbook: детальная обработка задач

Это расширение шагов 3, 6, 7 основного SKILL.md. Используй, если нужно уточнение формата, парсинга или обработки блокеров.

## Статусы change.md (шаг 3)

Прочитай `masterspec/changes/<name>/change.md` и проверь поле `> **Статус**:` в шапке. Реакция по значению:

| Значение | Действие |
|---|---|
| `Согласовано` | Обнови статус на **`В реализации`** (auto-переход, первый запуск). Больше ничего в change.md не трогай. |
| `В реализации` | Ничего не меняй, продолжай (повторный запуск, fluid workflow). |
| `Реализовано` | Сообщи, что изменения уже влиты, предложи `masterspec-archive-change`. Выйди. |
| `Архивировано` | Сообщи, что change завершён и заархивирован. Выйди. |
| `На согласовании` | **Блок**: сообщи, что change ещё не согласован, попроси завершить ревью и поставить статус `Согласовано` вручную. Выйди. |
| Другой / отсутствует | Предупреди о некорректном статусе, спроси подтверждение через **AskUserQuestion**. Если подтвердил — установи `В реализации`; иначе выйди. |

**Правило правки статуса**: меняй ТОЛЬКО строку `> **Статус**:` в шапке change.md. Остальные поля, разделы, таблицы — не трогай.

## Поиск первой невыполненной задачи (шаг 6)

**Область поиска — только пронумерованные группы.** Считай задачами строки `- [ ]` **только** в секциях, чей H2-заголовок начинается с цифры и точки (например `## 1. Подготовка`, `## 2. Реализация`, `## 3. Тестирование`).

**Игнорируй** всё содержимое секций, чей H2-заголовок НЕ начинается с цифры, вне зависимости от текста. Это покрывает:

- `## Чеклист верификации`
- `## Статус выполнения`
- `## Лог изменений`
- любые другие нечисловые H2-секции

**Правило парсинга:**

1. Проходи файл сверху вниз.
2. Запоминай текущий H2-заголовок.
3. Если заголовок соответствует regex `^##\s+\d+\.\s` — это рабочая секция, строки `- [ ]` внутри — задачи.
4. Иначе — секция вне области, пропускай все её `- [ ]`.
5. Первая встреченная задача `- [ ]` в рабочей секции — целевая.

Порядок внутри рабочих секций: сверху вниз.

## Цикл выполнения задачи (шаг 7)

Для каждой невыполненной задачи:

a. Покажи: `Выполняю X.Y: <описание>`
b. Реализуй строго то, что в задаче — без креатива, без расширения scope.
c. Сразу после выполнения замени `- [ ]` → `- [x]` в tasks.md.
d. Обнови прогресс: `Прогресс: N/M задач выполнено`.
e. Переходи к следующей задаче.

**Пауза (Output On Pause) если:**

- Задача непонятна → спроси.
- Реализация выявила проблему в `design.md` (если он есть) → предложи исправить.
- Реализация упёрлась в архитектурный вопрос, а `design.md` отсутствует → предложи запустить скилл `masterspec-impl-plan`.
- Ошибка или блокер → сообщи и жди решения.
- Пользователь прервал.
