# masterspec-recover — восстановить описание

Восстанавливаю описание из имеющегося материала. Не домысливаю: что не подтверждено источником — помечаю как открытый вопрос. Мета-модель и дисциплина слоёв — в kernel (`../masterspec/`).

## source=docs — из документов
1. Прочитай источники, составь карту: фрагмент → тип артефакта и слой.
2. Создай индекс-скелет (`templates/tpl-masterspec-index.md`).
3. Создавай артефакты от требований к спецификациям, в каждом укажи источник (`status: draft`).
4. Всё неоднозначное — в «Открытые вопросы». Не домысливай — фиксируй и спрашивай.
5. Собери глоссарий.

## source=code — из кода
1. Анализ структуры: языки, фреймворки, модули, корни (`roots=`). Используй `explore`.
2. Снизу: собери кодовый слой (`03-codemap/`): `repo-map`, `cmap-*` (компонент → код по `file:line`), `trace-*` (цепочка вызовов сценария), `dmap-*` (сущности → таблицы). Все — `generated: true`.
3. Вверх: по трассам и сигнатурам восстанови спецификации (компоненты с возможностями, сценарии, API, схемы данных) и требования (функции, концептуальную модель).
4. Всё восстановленное — `status: draft`, «Открытые вопросы: восстановлено из кода». То, что нельзя увидеть из кода (SLA, приоритеты, бизнес-правила) — вопросы бизнес-владельцу.

## Выход
- черновики артефактов (`status: draft`) с пометкой источника;
- индекс с разделом «Белые пятна»;
- список открытых вопросов для уточнения.

После восстановления — вычитка `verify` и доведение через `derive`/`evolve`.
