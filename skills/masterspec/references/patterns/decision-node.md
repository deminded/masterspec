# Узел-РЕШЕНИЕ (brainstorming-узел)
_Тип узла, где надо ВЫРАБОТАТЬ решение, а не исполнить готовое (Евгений msg 4948). По мотивам superpowers/brainstorming, адаптировано под masterspec (слои + резка/внедряемость). Парный к узлу-ИСПОЛНЕНИЕ (планировщик→исполнитель→приёмщик из element-workflow)._

## Два типа узла
| | узел-РЕШЕНИЕ | узел-ИСПОЛНЕНИЕ |
|---|---|---|
| когда | есть реальная развилка (несколько жизнеспособных вариантов) | вариант один / механический |
| ядро | дивергенция → выбор → DR | планировщик → исполнитель → приёмщик |
| выход | decision record (R-DR) + выбранный артефакт | артефакт + приёмка |

## Decision-worthiness (моя оговорка — когда вообще нужен)
Метод оценивает ДО запуска: есть ли развилка с разными компромиссами (архитектура, алгоритм, форма контракта, модель данных)? Да → узел-решение. Нет → прямое исполнение. Иначе дивергенция на тривиальном элементе = расход (против резки/внедряемости).

## Процесс узла-решение (адаптация brainstorming superpowers)
1. Контекст + критерии (что выбираем, ограничения, критерий «лучше»).
2. ДИВЕРГЕНЦИЯ: 2-3 варианта с компромиссами, рекомендуемый первым. Можно ансамблем субагентов с разными оптиками (наш Workflow-паттерн — ровно то, что делалось с архитектурой routes).
3. СУДЕЙСТВО: сравнить по критериям, держа напряжения.
4. ВЫБОР: один вариант или обоснованный гибрид.
5. DECISION RECORD (R-DR): зафиксировать ПОЧЕМУ — adr-* (решение фабрики) или dr-процесса (решение акта аналитики). Lazy-load.
6. HARD-GATE: человек согласует выбор до перехода к исполнению (enforcement).

## Рекурсия (Евгений: мастерплан + компоненты)
Применим на ЛЮБОМ уровне: скетч/мастерплан архитектуры (развилок больше всего) и отдельный компонент с реальным выбором. Тот же паттерн в миниатюре.

## Где удерживать, где снимать (перекличка dialog-ne-fusion + урок эксперимента)
Дивергенция УДЕРЖИВАЕТ противоречие как ресурс — но на выходе узла-решения оно СНИМАЕТСЯ в один выбор. Особенно контракт: single-source. Удержание — на фазе вариантов; снятие — в decision. НЕ путать продуктивную развилку (выбор с DR) с расщеплением (две несведённые редакции = баг, ловит приёмщик/O0).
