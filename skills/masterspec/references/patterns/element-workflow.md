# Micro-workflow элемента (планировщик → исполнитель → приёмщик)
_Уточнение гранулярности (Евгений msg 4945): у генерации КАЖДОГО элемента — свой внутренний воркфлоу, не один генератор. Рекурсия Role-Method-Work: тот же план/исполнение/приёмка на каждом уровне. Лечит расщепление из эксперимента wi2mchsxr (два исполнителя дали две редакции — приёмщика не было)._

## Три роли на один элемент
1. **Планировщик** — что генерить: какой артефакт, по каким требованиям/фокус-набору, какой шаблон, какие связи вверх. Выдаёт план элемента. (Если элемент сложный — может разбить на под-исполнителей.)
2. **Исполнитель** — генерит артефакт по плану в изолированном контексте. Может быть НЕСКОЛЬКО (напр. контракт большой) — тогда каждый делает часть.
3. **Приёмщик** — проверяет ИМЕННО этот элемент: самопроверка по оси O1; дозапрос ≤2 если не хватило; и — критично — **СВОДИТ редакции в одну (single-source) НА МЕСТЕ**, если исполнителей было несколько. Снимает разноголосие до того, как оно дойдёт до гейта слоя.

## Почему приёмщик обязателен (урок эксперимента)
В прогоне контракт писали 2 исполнителя (структурный + контрактный) без приёмщика → две несведённые редакции доехали до слоя → конфликт (опаснее вакуума). Приёмщик элемента = консолидация на месте: дешевле и точнее, чем ловить расщепление гейтом слоя. Контракт — место, где разноголосие СНИМАЕТСЯ (fusion), не удерживается.

## Рекурсия по уровням
| Уровень | Планировщик | Исполнитель | Приёмщик |
|---|---|---|---|
| Фабрика | (человек/аналитик) | route-скиллы | человек-гейт |
| Слой (Route A/B) | ОРКЕСТРАТОР (route-скилл) | элемент-генераторы | ревьюер слоя (review-gate) + негатив |
| Элемент | планировщик элемента | gen-* исполнитель(и) | приёмщик элемента (single-source + O1) |

ВАЖНО (уточнение ролей, msg 4945): РАЗВЕДЧИК (explore) ≠ ОРКЕСТРАТОР. Разведчик поставляет контекст (read-only). Оркеструет генераторов route-скилл (оркестратор) — отдельная явная роль, на карте была неявной («дорожкой»).
