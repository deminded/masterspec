# metadata

- **Название:** f-desc
- **Автор:** Парфёнов
- **Версия:** 1.0.0
- **Описание:** Скилл описывает фабрики (автоматизированные системы / функциональные продукты) по трёхслойной мета-модели: требования → спецификации → кодовая база. Обеспечивает сквозную трассировку «что хотим → как устроено → где в коде», без дублирования между слоями.
- **Применимость:** Используйте, когда нужно спроектировать новую фабрику, восстановить описание из существующих документов, сгенерировать codemap по репозиторию, провести аудит полноты описания или выполнить reverse engineering. Подходит для работы с паспортами АС/ФП, функциями, НФТ, компонентами, сценариями, алгоритмами, API, схемами данных, диаграммами C4, профилями нагрузки, справочниками, тест-кейсами и ADR.
- **Использование:** Положите директорию `F_DESC` в `~/.claude/skills/` (или эквивалент для GigaCode). Вызывайте через естественный запрос («опиши фабрику X», «сделай codemap репозитория», «проведи аудит описания фабрики Y») либо явно укажите режим: `design` / `recover` / `codemap` / `audit` / `reverse`. Скилл автоматически определит нужные шаблоны из `templates/`, мета-модель из `meta_model.md` и границы слоёв из `SKILL.md` §0.
- **Эксперимент:** До этого описания фабрик расползались по слоям — функциональные требования смешивались с названиями компонентов, спецификации с именами классов, а трассировка до кода отсутствовала. Скилл вводит жёсткую дисциплину границ слоёв (явные запреты по каждому слою), bottom-up направление ссылок, capabilities (`cap-*`) как стыковку функций и кода, а также два уровня тест-кейсов (`tc-acc-` на требованиях, `tc-int-` на спецификациях).
