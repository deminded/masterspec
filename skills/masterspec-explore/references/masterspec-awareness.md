# Осведомлённость об артефактах masterspec

Аналог `openspec-awareness.md` — как лид-скил (`masterspec-explore` или его вызывающий) должен сориентироваться в директории фабрики перед запуском ресеча.

---

## 1. Проверка наличия фабрики

Перед началом работы выполни:

```bash
ls masterspec/ 2>/dev/null
```

Ориентируйся на маркеры:

| Признак | Что значит |
|---------|-----------|
| Есть `masterspec/00-masterspec-index.md` | Фабрика уже описана. Read его — получишь карту существующих slug'ов по слоям. |
| Есть `masterspec/01-requirements/` | Частично описаны требования. |
| Есть `masterspec/02-specifications/` | Есть спецификации компонентов/сценариев. |
| Есть `masterspec/03-codemap/` | Есть codemap — можно переиспользовать как стартовую точку. |
| Есть `masterspec/changes/` | Есть история изменений. Проверь `changes/<name>/` — возможно, есть активный change. |
| Ничего из вышеперечисленного | Фабрика ещё не описана — `target=factory-spec` должен создавать артефакты с нуля через вызывающий скил. |

Если `masterspec/` отсутствует — предложи вызывающему скилу сначала запустить kernel-скилл `masterspec` в режиме `design` (чтобы создать индекс и базовые артефакты).

---

## 2. Активные changes

```bash
ls masterspec/changes/ 2>/dev/null | grep -v "^archive$"
```

Для каждого активного change прочитай шапку `change.md` и возьми статус. Если пользователь запустил исследование без явного указания — предложи уточнить:

| Ситуация | Вопрос к пользователю |
|----------|-----------------------|
| Есть один активный change в статусе `Согласовано`/`В реализации` | «Картируем зону этого change?» (`target=factory-change`, `name=<имя>`) |
| Несколько активных changes | «Какой change картируем?» — перечислить имена и статусы |
| Нет активных changes | «Картируем всю фабрику или конкретную зону?» (`target=factory-spec` vs `factory-change` с явным anchor) |

---

## 3. Переиспользование существующего codemap

Если в `masterspec/03-codemap/` уже есть артефакты (`repo-map`, `cmap-*`, `trace-*`, `dmap-*`), прочитай `00-repo-map.md` и **не перезапускай `codemap-scope`** полностью. Запусти его только в режиме «обновить» (передай существующие slug'и в `anchor_paths`).

Обрати внимание на поле `updated:` в YAML-фронтматтере — если старше 90 дней, предупреди пользователя: «Codemap старый, код мог уйти вперёд, предлагаю перезапустить».

---

## 4. Свежий `.research-notes.md`

Если `.research-notes.md` уже есть рядом с целевым артефактом:

- `masterspec/.research-notes.md` для `target=factory-spec`
- `masterspec/changes/<name>/.research-notes.md` для `target=factory-change`

Проверь `mtime`:

```bash
stat -c '%Y' masterspec/.research-notes.md 2>/dev/null || stat -f '%m' masterspec/.research-notes.md 2>/dev/null
```

Если не старше 24 часов — переиспользуй, субагентов не запускай повторно. Просто верни путь к `.research-notes.md` и краткую сводку.

Если старше 24 часов — сообщи пользователю и спроси, перезапускать или переиспользовать.

---

## 5. После исследования: что делать с результатом

Лид-скил (`masterspec-explore` standalone) возвращает пользователю:

1. Путь к `.research-notes.md` + `.research/_aggregate.yaml`.
2. Краткую сводку (N fn-кандидатов, M cmp-кандидатов, K scn-кандидатов, L apis, P data_schemas).
3. Список `gaps[]` как приоритизированные вопросы для Этапа 2 интервью.

Затем предлагает следующий шаг:

| Ситуация | Следующий скил |
|----------|----------------|
| `target=factory-spec`, фабрики ещё нет | `masterspec` в режиме `design` (использует `.research-notes.md` как черновой материал) |
| `target=factory-spec`, фабрика есть, но не актуальна | `masterspec` в режиме `audit`/`reverse` |
| `target=factory-change` | `masterspec-propose` (создаст change.md, подхватив `.research-notes.md`) |

---

## 6. Запреты

- **Не создавай артефакты фабрики.** Единственное, что пишет `masterspec-explore`, — это файлы в `.research/`.
- **Не удаляй старые `.research/`** без подтверждения пользователя — там могут быть несохранённые инсайты.
- **Не читай `masterspec/changes/archive/`** без явного запроса — это исторический аудит, не источник текущей картины.
