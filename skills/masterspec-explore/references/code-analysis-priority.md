# Приоритет инструментов анализа кода

Когда скилл требует понять существующий код (пакеты, символы, зависимости, вызовы), всегда выбирай **наиболее семантический** из доступных инструментов. Грубые текстовые средства — последняя линия.

## Порядок приоритета

1. **Serena (MCP)** — `find_symbol`, `get_symbols_overview`, `find_referencing_symbols`. Семантический анализ, точная навигация по символам, поиск всех мест использования.
2. **LSP-инструменты** — go-to-definition, find-references, workspace symbols. Работают, если в среде поднят языковой сервер (gopls, pyright, jdtls и т. д.).
3. **Code index / embeddings** — семантический поиск по кодовой базе (например, через MCP-сервер индексации).
4. **Другие MCP-серверы** — любые специализированные инструменты анализа, навигации, индексации, которые доступны в сессии.
5. **Grep / Glob** — текстовый поиск. Используется только когда ни один из семантических инструментов не подходит или не доступен.

## Правила применения

- **Не ограничивайся чтением отдельных файлов.** Строй полную картину: зависимости, вызывающий код, связанные сущности.
- **Не деградируй преждевременно.** Если Serena доступна — используй её; не падай сразу в grep только потому, что «так быстрее».
- **Комбинируй.** Семантический инструмент + точечный grep по литералам (строки, регулярки, магические константы) — рабочая связка.
- **Фиксируй контекст.** Прежде чем принимать решение по change.md / design.md, убедись, что изучены все точки интеграции, вызывающий код и конфигурация.
