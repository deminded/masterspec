## Чек-лист соответствия tc
- [ ] Название краткое, без нумерации и отражает путь сценария.
- [ ] `criticality` согласована с функцией; нет понижения критичности теста.
- [ ] Кейс основан только на scn/api/tc-flt; неподтверждённых ошибок и дублей нет.
- [ ] Предусловия атомарны; каждый шаг содержит действие, testData и expectedResult.
- [ ] Действие начинается с глагола в повелительном наклонении и атомарно.
- [ ] expectedResult конкретен и верифицируем на заявленной границе.
- [ ] testData конкретны либо содержат явное обоснованное N/A.
- [ ] Отказный кейс ссылается на стабильный `tc-flt/FLT-*`.
- [ ] Для ошибки/сбоя есть отдельная проверка логов с correlation-id/окном/событием.
- [ ] Логи дополняют пользовательский долговечный след `OE-EVIDENCE`, а не заменяют его.
- [ ] Аудит и мониторинг проверяются, если они заявлены в scn/OE.
