## Чек-лист соответствия tc
- [ ] Название краткое, без нумерации и отражает проверяемое условие.
- [ ] `criticality` согласована с функцией; нет понижения критичности теста.
- [ ] Проверяются только AC/OE и ошибки из аналитики; новых сценариев и дублей нет.
- [ ] Предусловия атомарны; каждый шаг содержит действие, testData и expectedResult.
- [ ] Действие начинается с глагола в повелительном наклонении и проверяет один переход.
- [ ] expectedResult конкретен и верифицируем, без общих фраз.
- [ ] testData конкретны либо содержат явное обоснованное N/A.
- [ ] Для UI авторизация оформлена отдельным первым шагом, если входит в путь.
- [ ] Для ошибки/сбоя есть отдельная проверка логов с correlation-id/окном/событием.
- [ ] Проверка логов дополняет, а не подменяет oracle `OE-EVIDENCE`.
- [ ] Аудит и мониторинг проверяются, если они заявлены в fn/OE.
