# masterspec-accept — принять слой (промоушен + reindex под контролем)

Закрываю СЛОЙ (`req`|`spec`) после вычитки/разрешения незакрытых вопросов. НЕ генерирую и НЕ правлю
содержание (это `derive`/`evolve`/`gen`) — перевожу ЗРЕЛОСТЬ артефактов (draft→actual) и синхронизирую
индекс. Снимаю ручную боль: «менять статус каждому + reindex руками — издевательство».

## Вход
- **`layer=req|spec`** — какой слой закрываем (пары́тся с `derive layer=`: derive породил — accept
  закрывает тот же объект).
- **`factory=`** — путь к фабрике/репо со SSoT (по умолчанию — текущая).

## Шаги
1. **Гейт остатка (детерминированно, не на глаз).**
   - Прогнать form-detector: `../masterspec/scripts/check-operational-envelope.py <factory> --scope
     <layer>` → OPEN-грани, form-TODO (`MIGRATE-TODO`/`x-migrate-todo`), дыры O_T, непройденные оси.
   - Скан незакрытых вопросов/маркеров по артефактам слоя: `OPEN`, `TC-TODO`, «Открытый вопрос»,
     литеральные заглушки (`TBD`/`TODO`/`…`), незакрытые узлы-решения.
   - Итог — СПИСОК остатка (артефакт → что открыто), а не впечатление.
2. **Контроль человека.** Показать остаток; по каждому пункту — развилка (`AskUserQuestion`):
   «закрыть сейчас» (вернуть на доработку, промоушен не делать) ИЛИ «отложить» → записать в §7 «Белые
   пятна» Паспорта/индекса как явный TODO. **Ничего с открытым блокером не метится actual молча.**
3. **Промоушен.** Артефакты слоя без открытого блокера (или с явно отложенным) → `status:
   draft→actual` во фронтматтере. Отложенные ОСТАЮТСЯ `draft` + записанный TODO. `adr-`/`dr-` ведут
   свой решенческий цикл — их статус НЕ трогаю.
4. **Reindex.** Полная перегенерация `00-masterspec-index.md` по `../masterspec/references/index-canonical.md`
   §5 — отражает новые статусы (единственный способ; точечные правки строк не применяются).
5. **Сертификат (если применимо).** Весь применимый слой зелёный и без OPEN → штамп
   `meta_model_version` в Паспорт (`../masterspec/references/migration.md` §5 / `apply-change` §11).
   Красный/отложенное → штамп НЕ ставится, отчёт «слой остаётся до-X, остаток: …».

## Два пути (сходятся в этом скилле)
- **Путь 1** — человек сам вычитал, поправил и поменял статусы. accept = гейт остатка + reindex
  (+ сертификат): подтвердить, что открытого не осталось, и перестроить индекс.
- **Путь 2** — незакрытые вопросы разрешены с агентом. accept на «подтверждаю вычитку слоя»: гейт →
  промоушен draft→actual → reindex.

## Границы
- Не генерирую и не правлю СОДЕРЖАНИЕ артефактов (derive/evolve/gen); accept меняет только статус +
  индекс.
- Не снимаю `OPEN`/блокер молча — только явным отложением человека в §7.
- `adr-`/`dr-` статусы не трогаю (свой цикл).
- Индекс — только полной перегенерацией, не точечно.

## Композиция
`recover`/`derive layer=<l>` дают слой в `draft` + незакрытые вопросы; человек/агент их разрешают;
`verify scope=<l>` вычитывает; **accept** закрывает слой (гейт → промоушен → reindex → сертификат).
Гейт остатка — тот же form-detector, что у `verify`/`migration`.
